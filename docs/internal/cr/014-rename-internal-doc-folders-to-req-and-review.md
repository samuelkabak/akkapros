# Change Request: Rename Internal Documentation Folders to req and review

CR-ID: CR-014
Status: Done
Priority: Medium
Impact: Additive
Created: 2026-03-25
Updated: 2026-03-25
Implements: None
---

# Summary

Rename the internal documentation folders `docs/internal/specs` to `docs/internal/req` and `docs/internal/reviews` to `docs/internal/review`.

The current mix of plural folder names is inconsistent with the existing singular `adr` and `cr` folders. This change makes the internal documentation layout coherent, shortens the path names, and removes ambiguity between requirements (`req`) and informal specifications.

---

# Motivation

The current internal documentation tree mixes naming styles:

- `adr/`
- `cr/`
- `specs/`
- `reviews/`

This inconsistency leaks into contributor guidance, generated indexes, and cross-links throughout the repository. Renaming the requirements and review directories to singular forms aligns the naming convention across all internal record folders and simplifies tooling assumptions.

---

# Scope

## Included

- Rename `docs/internal/specs/` to `docs/internal/req/`
- Rename `docs/internal/reviews/` to `docs/internal/review/`
- Update index-generation logic in `scripts/update-indexes.py`
- Update internal and public documentation links that reference the old paths
- Regenerate affected internal index files after the move

## Not Included

- Renaming ADR or CR folders
- Changing requirement or review filename conventions beyond directory paths
- Rewriting the substance of existing requirement or review documents

---

# Current Behavior

Internal documentation currently stores requirements under `docs/internal/specs/` and reviews under `docs/internal/reviews/`.

Repository documentation, contributor guidance, and index-generation tooling all reference these plural directory names directly.

---

# Proposed Change

- Use `docs/internal/req/` for requirements
- Use `docs/internal/review/` for reviews
- Preserve existing file names inside those folders
- Keep autogenerated indexes in the renamed folders
- Update all tracked documentation and scripts to the new canonical paths

---

# Technical Design

Implementation steps:

Components:
- Internal documentation tree under `docs/internal/`
- Index generator in `scripts/update-indexes.py`
- Contributor and project documentation that links to internal records

Directory changes:
- `docs/internal/specs/` → `docs/internal/req/`
- `docs/internal/reviews/` → `docs/internal/review/`

Script updates:
- Replace `SPEC_DIR` with `REQ_DIR`
- Replace `REVIEWS_DIR` with `REVIEW_DIR`
- Update index builder names and error messages to the new canonical directories

Documentation updates:
- Replace references to `specs` with `req` where they denote the path
- Replace references to `reviews` with `review` where they denote the path
- Preserve semantic wording such as “requirements/specs” where it refers to document type rather than filesystem path
