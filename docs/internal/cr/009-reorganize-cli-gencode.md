# Change Request: Reorganize CLI common code and add `_gencode` generators

CR-ID: CR-009
Status: Draft
Priority: Medium
Created: 2026-03-20
Updated: 2026-03-20
---

# Summary

Move non-CLI library code out of the `cli/` package into `lib/`, introduce
a dedicated `_gencode` package for code generators, and update the build
generation configuration and documentation. This CR documents the planned
repository reorganization; implementation will be done in follow-up work.

---

# Motivation

`src/akkapros/cli/` must only contain CLI entrypoints and thin wrappers.
Utility code and business logic that is reusable across CLIs belongs in
`src/akkapros/lib/`. Separately, code generators (scripts that emit
static/generated artifacts for the runtime) should be grouped under a
top-level `_gencode` package to separate generator source from runtime
library code.

This separation improves maintainability, clarifies public APIs vs
generator scripts, and makes it easier to publish or import `lib/` code
without pulling generator code into distributions.

---

# Scope

Included:

- Move `src/akkapros/cli/_cli_common.py` → `src/akkapros/lib/utils.py`.
- Create a new folder `src/akkapros/_gencode/` and place generator modules
  there.
- Move `src/akkapros/_gen_diphthongs.py` →
  `src/akkapros/_gencode/lib_diphthongs.py`.
- Modify the diphthong generator so it writes the target `lib/diphthong.ph`
  file directly and inserts a top comment indicating the generated file is
  autogenerated from `src/akkapros/_gencode/lib_diphthongs.py`.
- Add a `[tool.build]` `generate` entry (or equivalent build hook) so that
  generators can be run from the build system. Example entry to be added to
  `pyproject.toml` or equivalent:

```
[tool.build]
generate = [
    "python -m src.akkapros._gen.lib_diphthongs"
]
```

Note: the module path above follows the current generator discovery in the
request; when implementing, ensure the module path matches the created
package name (`_gencode` vs `_gen`).

Also included: add documentation and brief reference entries describing the
new `_gencode` folder and regeneration steps in developer docs (e.g.
`docs/`, `CONTRIBUTING.md`, and `docs/internal/adr` as appropriate).

---

# Current Behavior

Shared utility code currently lives under `src/akkapros/cli/_cli_common.py`.
Generator code such as `_gen_diphthongs.py` resides at top-level `src/
akkapros/` and mixes generator logic with runtime modules. There is no
dedicated `_gencode` package or documented regeneration workflow.

---

# Proposed Change

1. Move `src/akkapros/cli/_cli_common.py` into `src/akkapros/lib/utils.py`.
   Update imports across the repository to use `akkapros.lib.utils`.

2. Create `src/akkapros/_gencode/` and add an `__init__.py` so it is a proper
   package. Move generator scripts under this package; specifically move
   `src/akkapros/_gen_diphthongs.py` →
   `src/akkapros/_gencode/lib_diphthongs.py`.

3. Update the diphthong generator to write its output file directly to
   `src/akkapros/lib/diphthong.ph` and add a generator comment header such
   as:

```
# This file is autogenerated by src/akkapros/_gencode/lib_diphthongs.py
# Do not edit directly.
```

4. Add a build-time generator hook so maintainers can run generators from the
   project's build tooling. The CR documents the example `[tool.build]`
   `generate` entry; the implementation should ensure the module path is
   correct (matching `_gencode` package name) before adding to `pyproject.toml`.

5. Add developer-facing documentation describing the `_gencode` layout and
   the regeneration workflow. Mention where generated outputs go, and how to
   run the generators locally.

---

# Technical Design

- `src/akkapros/lib/utils.py` should expose the functions previously in
  `_cli_common.py` under namespaced `akkapros.lib.utils`.
- Generators in `_gencode` are normal Python modules that can be executed
  with `python -m src.akkapros._gencode.lib_diphthongs` (or the correct
  module path chosen during implementation).
- Generated files must contain a single-line header comment indicating the
  generator source and a timestamp or version string if available.
