# Summary

This change request has been implemented.

Implemented scope:

- Moved shared CLI helpers out of `src/akkapros/cli/` and into `src/akkapros/lib/utils.py`.
- Updated all CLI imports from `akkapros.cli._cli_common` to `akkapros.lib.utils`.
- Removed `src/akkapros/cli/_cli_common.py`.
- Created `src/akkapros/_gencode/` with `__init__.py`.
- Moved diphthong generator from `src/akkapros/lib/_gen_diphthongs.py` to `src/akkapros/_gencode/lib_diphthongs.py`.
- Updated generator output header to reference `src/akkapros/_gencode/lib_diphthongs.py`.
- Added a build generator hook in `pyproject.toml`:

```toml
[tool.build]
generate = [
    "python -m akkapros._gencode.lib_diphthongs",
]
```

- Updated developer documentation for the new `_gencode` workflow.
- Added a generator smoke test.

---

# Motivation

`src/akkapros/cli/` should contain CLI entrypoints and thin wrappers only.
Reusable logic belongs in `src/akkapros/lib/`.
Code-generation scripts should be grouped separately under `src/akkapros/_gencode/`
so runtime code and generator code are clearly separated.
