# v2.0.0 — Breaking interface cleanup, validation hardening, and release tooling

**Release date:** 2026-03-23

## Summary

This major release consolidates the post-1.0.1 change set into a coherent new
baseline. It includes external naming changes, escape-syntax changes, stricter
input validation, stronger regression coverage, internal tooling improvements,
and a restoration of ADR-002 centralized version management.

## Breaking Changes

- JSON/CSV field names previously using `repair*` now use `accentuated*`.
- Escaped foreign-text syntax now uses `{{text}}` and `{tag{text}}` instead of
  legacy bracket-based forms.
- Input validation is stricter in file-input CLIs and punctuation handling is now
  governed by an allowlist.

## Highlights

- Added an end-to-end integration test with pinned gold-standard metrics.
- Added format-validation guards for file-input CLIs, including a conservative
  Akkadian-likelihood check for `*_proc.txt` inputs.
- Reorganized shared CLI logic into `src/akkapros/lib/` and expanded helper
  tooling under `scripts/`.
- Refactored `phoneprep` core and filename handling.
- Improved internal project structure with ADR/CR/spec/review indexes and
  additional contributor documentation.
- Restored centralized version management so packaging derives the release
  version from `src/akkapros/__init__.py`.

## Upgrade Notes

- Update downstream parsers that still read `repair*` JSON/CSV keys.
- Convert any legacy escaped-text markup before processing older material.
- Re-test custom punctuation-heavy workflows against the validation allowlist.
- If building from source, use `python scripts/build_package.py` so packaged
  documentation is synced before building.

## Validation

Recommended release checks:

```bash
python scripts/sync_docs.py
python scripts/build_package.py --sdist --wheel
pytest -q
```

Optional CLI self-tests:

```bash
python -m akkapros.cli.fullprosmaker --test-all
python -m akkapros.cli.printer --test
```
