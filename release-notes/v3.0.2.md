# v3.0.2 — LOB circumflex documentation clarification

**Release date:** 2026-05-14

## Summary

This patch release clarifies the LOB accentuation documentation to explain how
circumflex vowels (â, ê, î, û) are treated as superheavy for stress purposes
when they appear in final position. It is the second maintenance release on the
v3.x.x series, which introduced the phonetizer stage, a complete metrics
redesign, and a package-wide YAML configuration system.

## Documentation Fix

- **LOB circumflex vowel rule now documented.** The prosody realization algorithm
  doc (`docs/akkapros/prosody-realization-algorithm.md`) previously listed only
  `CVVC` and `VVC` as superheavy syllable shapes. It now includes a note
  explaining that a final CVV or VV syllable whose vowel is circumflex is treated
  as superheavy (3 morae) and therefore attracts stress under the LOB
  final-superheavy rule. Non-final circumflex syllables do not automatically
  attract stress; they compete under the standard rightmost-eligible rule. The
  implementation in `src/akkapros/lib/prosody_model.py` was already correct; only
  the documentation was incomplete.

- Full phoneme framework with MBROLA/X-SAMPA `.pho` output
- Duration modeling with configurable global scale
- Intonation contours with pause-governed synchronization
- Pause typing (short/long) with drift tracking
- Probability-oriented diagnostics and ratio-preserving shortfall carry
- Pre-pausal final anchors and corrective geminate coda share ratio
- Class-local gemination caps and experimental feature guard
- Proto-Semitic pharyngeal/glottal replacement (`ḥ/ʿ/ʾ` → `ʔ`)
- Ultraheavy hiatus marker phone and MBROLA output

- Interval-based metrics from phone/ophone pairs
- Per-row drift columns in phone files
- Pause-governed intonation with half-beat synchronization
- VarcoC, VarcoV, rPVI-C, nPVI-V, %V, %C, ΔC, ΔV
- Speech rate (WPM), pause ratio, articulation duration
- JSON and table output formats

### Configuration System

- `confwriter` for config management and verification
- Scoped `--help [PATH]` for targeted documentation
- Effective runtime config model

### Pipeline Enhancements

- Punctuation armor (`⟦...⟧`) preserved through all pipeline stages
- Emphatic vowel coloring extended to coda contexts and phoneprep
- Configurable `limit_emphatic_coloring` with inverted default
- Business-zone module splitting for LLM cost optimization (CR-087, CR-091,

- Code index with Mermaid call graphs and tagged test markers

### Documentation

- Mermaid flowcharts in user-facing documentation
