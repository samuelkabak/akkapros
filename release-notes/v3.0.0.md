# v3.0.0 — Phonetizer realization, metrics redesign, and config-driven pipeline

**Release date:** 2026-05-10

## Summary

This major release introduces the phonetizer as a full pipeline stage with
MBROLA/X-SAMPA output, a complete metrics redesign around phone/ophone interval
metrics, a package-wide YAML configuration system, and extensive internal
restructuring for maintainability. It represents approximately 100 CRs of
cumulative changes since v2.0.0.

## Breaking Changes

- **Phonetizer output format**: `.pho` output now uses MBROLA/X-SAMPA-like
  symbols derived from the realization inventory instead of internal realization
  codes. The residual printer MBROLA renderer has been removed.
- **Phone/ophone row separator**: unified to `|`; row-level drift tokens now use
  signed arithmetic form `[+-]DDD` (e.g. `-012`, `+000`, `+054`).
- **Metrics redesign**: metrics now operate on phone/ophone interval data rather
  than tilde pivot. CSV output has been removed; JSON and table formats remain.
- **Config restructuring**: stage configuration is now split into `run` and
  `process` blocks. The `--conf` and `--option` system replaces many individual
  CLI flags. The `confwriter` CLI manages config files.
- **Removed CLI options**: `--ipa-proto-semitic` (printer), `--long-pause-weight`
  (metrics), short-pause and drift policy options, `--csv` output flag.
- **Renamed options**: `print.run.circ_hiatus` → `print.run.ipa_ultraheavy_hiatus`,
  `extended_emphatic_coloring` → `limit_emphatic_coloring` (inverted default),
  `mono_mode_accentuation_lengthening` → `basic_accentuation_lengthening`.
- **Merge connectors**: explicit inherited merges (`+`) are now distinguished
  from internal prosody merges (`&`) in the tilde pivot.
- **Punctuation armor**: `⟦...⟧` sequences are preserved through the tilde pivot
  and downstream stages.

## Highlights

- **Phonetizer stage**: full phoneme framework with duration modeling, intonation
  contours, pause typing, drift tracking, and MBROLA `.pho` export.
- **Metrics redesign**: interval-based metrics from phone/ophone pairs with
  per-row drift columns, pause-governed intonation, and half-beat synchronization.
- **Config system**: package-wide YAML config with `--conf`/`--option` CLI,
  `confwriter` for config management, scoped `--help [PATH]`, and effective
  runtime config model.
- **Internal restructuring**: business-zone module splitting (CR-087, CR-091,
  CR-092), `_metrics_stats.py`/`_metrics_output.py` split, code index with
  Mermaid call graphs, and tagged test markers for optimized execution.
- **Phonetizer enhancements**: global duration scale, pre-pausal final anchors,
  corrective geminate coda share ratio, resync pause toggle, probability-oriented
  diagnostics, ratio-preserving shortfall carry, class-local gemination caps,
  and experimental feature guard.
- **Emphatic coloring**: extended to coda contexts and phoneprep; configurable
  `limit_emphatic_coloring` with inverted default.
- **Proto-Semitic replacement**: phonetizer now supports configurable
  pharyngeal/glottal replacement (`ḥ/ʿ/ʾ` → `ʔ`).
- **Ultraheavy hiatus**: marker phone and MBROLA output for ultraheavy syllables.
- **Documentation**: comprehensive phonetizer docs (algorithm, data model, phone
  file guide), Mermaid flowcharts in user-facing docs, metrics coverage matrix,
  and expanded governance records.
- **Demo expansion**: config-driven prosmaker demo with explicit defaults,
  lexlinks construct demo with mono-mode branch.

## Upgrade Notes

- Update downstream parsers that consume `.pho` files to expect X-SAMPA-like
  symbols instead of internal codes.
- If you use `_phone.txt` or `_ophone.txt` programmatically, update row parsers
  for the `|` separator and signed drift tokens.
- Migrate any CLI scripts using removed flags (`--ipa-proto-semitic`, `--csv`,
  etc.) to the config system or their replacements.
- Review config files: stage settings now live under `run`/`process` blocks.
  Use `confwriter --verify` to check existing configs.
- If you rely on metrics CSV output, switch to JSON or table format.
- The printer no longer produces MBROLA output; use the phonetizer stage instead.
